## 头牌案例：一个会随你汇报而自动推进的项目

这是普通待办工具做不到的一步。你交给它的不是一条整理好的任务，而是一个**意图**。它识别出这是一个多步的**项目**，设定*正确的第一步*；等你带着进展回来，它**销掉旧的下一步、推导出新的下一步**——让项目一直活着，而你不用重新规划。

**第一轮** —— 一个模糊的意图：

> 你：
```
/gtd 我要培养我家女儿的乒乓球
```
> LLM-GTD：
```
已按 GTD 归位。

我把它建成了一个项目：📁 培养女儿的乒乓球兴趣与训练路径

第一步放在 Next Actions 的 @家：
  和女儿聊 10 分钟，确认她是否想尝试乒乓球、已有水平、
  每周能接受的练习频率和喜欢/不喜欢的方式。

我没有报名、排课、联系教练或写日历。
GTD 状态已验证：Inbox 0，Projects 3，Next Actions 3，stalled 0。

补充：GTD 编排器建议后续可加一条 @电脑「整理附近 3 个儿童乒乓球训练选择」。
我的判断是先问女儿意愿，再做机构调研。
```
*它没有直接跳到报名。一个关于「人」的项目，正确的第一步是先跟这个人聊——而且它有意把机构调研那一步往后压。*

**第二轮** —— 过了一会儿，你回来汇报：

> 你：
```
/gtd 我跟我女儿聊完了，她对乒乓球很有兴趣，每周可以上两次课每次课 1 小时
```
> LLM-GTD：
```
已理清并落盘。

你之前那条「和女儿聊 10 分钟」已从下一步里销项；项目推进到下一步：
  筛选 2-3 个适合女儿的乒乓球课，记录每周两次、每次约 1 小时的时段、
  费用、距离和试课方式。
  → next-actions.md

项目也更新了支持材料：女儿有兴趣、可接受每周两次课；你写的「1 小时」
我先按「1 小时」理解，但标成待确认。
  → projects.md + reference.md

检查结果：Inbox 0，Projects 3 且没有 stalled。
```
*项目自己往前走了一步。已完成的动作被销掉，新的下一步是从你新得到的信息里推导出来的，支持材料记下了事实，而推断出来的细节（「约 1 小时」）被标为待确认而非默认。*

**为什么这很关键**：一个没有当前下一步的项目，就是一个卡住的承诺。这里 harness 跨天把承诺一直挂活、替你做记账，而你只需提供判断、汇报现实。

## 1. 捕捉 → 自动理清（单条）

> 你：
```
/gtd-capture 出行前把护照续签了
```
> LLM-GTD：
```
已记并理清：「出行前把护照续签了」
→ projects.md「护照续签」（成果：有效护照在出行前到手）
→ next-actions @外出：查最近的受理点与所需材料
要改归类/动作就说一声。
```
*先落盘（不丢），再理清一条并归位。多步成果变成项目 + 一个具体下一步。*

## 2. 清空大脑（批量 → 先全捕捉，再批量理清）

> 你：
```
帮我清空大脑
```
> LLM-GTD 用触发清单把一切丢进 `inbox.md`，**不逐条打断**，然后：
```
捕捉了 14 项到 inbox。要我现在逐条理清吗？（批量）
```
*高量捕捉零摩擦；理清作为之后的批量动作。*

## 3. 执行 —— 现在该做什么

> 你：
```
/gtd-engage 我有 30 分钟，精力低，在电脑前
```
> LLM-GTD（先静默跑一遍 organize 结构卫生，再）：
```
@电脑 · 30 分钟 · 低精力 —— 现在最值得做：
1. 回邮件确认下周评审时间（~5 分钟）
2. 把护照材料清单整理进 reference（~10 分钟）
3. 草拟周报开头三段（~20 分钟）
（今天硬约定：16:00 团队同步 · 该催：上周委派的数据还没回）
```
*按情境 + 时间 + 精力筛，优先级对照 Horizons 校准。你来选。*

## 4. 回顾 —— 每周的关键一步

> 你：
```
/gtd-review
```
> LLM-GTD 生成只读备料包、清掉机械漂移，然后带你走：
```
── 系统全景 ── inbox 0 · next-actions 9 · projects 4（stalled 0）· waiting 2 · someday 6
① Get Clear   ✅ inbox 已清空
② Get Current 销项 3；该催 1；项目都挂着有效下一步
③ Get Creative someday「学钢琴」看起来还不到启动时机，保留
④ Horizons    某项目不服务于本季任何目标 → 建议砍或移 someday
下周聚焦 3 件：①… ②… ③…
```
*AI 备料、清理；**反思由你做**——这部分外包不出去。*

---

## 哪些留给了人

捕捉、理清、机械 organize 都自动化了。**执行**（现在做什么）和**回顾**（方向对不对）按设计留在你手里——而这正是 GTD 价值所在。
