# GTD Session Close Template

用于“本轮 session 状态 / 本轮 session 收尾 / 今天收尾”。目标不是写日报，而是做承诺系统 diff。

## 执行顺序

1. 扫描本轮会话，只抽取 GTD-relevant deltas：已完成并已写入的改动、新承诺、项目状态变化、阻塞、等待别人、硬日期、需要用户拍板的判断、未入 GTD 的知识洞察。
2. 对每条 delta 跑 clarify 决策树：需要行动吗？下一步是什么？属于哪个清单？知识洞察是否该走 ZK？
3. 需要落盘的先按 capture→clarify 写入 `memory/gtd/`，写完从 inbox 删除；不确定承诺不替用户拍板。
4. 收尾前跑 `scripts/gtd_status.sh`，用真实 dashboard 数字，不凭记忆报数。
5. 按下面五段输出；无内容写“无”，不要省略标题。

## 输出格式

**已完成并已落盘**
- ...

**新增 / 变更的 GTD 条目**
- Projects：...
- Next Actions：...
- Waiting For：...
- Calendar：...
- Product Ideas / Reference：...

**需要用户拍板的事项**
- ...

**未入 GTD 的知识洞察**
- ...

**当前 dashboard**
- Inbox：...
- Next Actions：...
- Projects：...（stalled：...）
- Waiting For：...
- Calendar：...
- Someday/Maybe：...
- Product Ideas：...

## 质量线

- 不把整段聊天总结塞进 `inbox.md` 或 `reference.md`。
- 不把“已经说明过”当作“已经落盘”；只有文件实际变更才算已落盘。
- 不为显然已完成的事项制造下一步。
- 不把知识洞察污染到 GTD 清单；必要时只提示“应走 ZK 管线”。
